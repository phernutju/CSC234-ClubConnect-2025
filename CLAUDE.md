## Project

ClubConnect — a Flutter mobile app for real-time community chat rooms with AI-powered content moderation. Built for CSC231/CSC234 at KMUTT. Targets Android, iOS, and Web.

## Commands

```bash
# Install dependencies
flutter pub get

# Run (Android default, or specify target)
flutter run
flutter run -d chrome

# Lint / static analysis
flutter analyze

# Tests
flutter test
flutter test test/widget_test.dart   # single test file

# Firebase setup (one-time; generates lib/firebase_options.dart which is gitignored)
firebase login
flutterfire configure
```

## Architecture

Layered: **UI → Providers → Services → Firebase / Gemini**

No screen imports Firebase or Gemini directly — all external calls go through the service layer.

```
lib/
├── main.dart                # App entry: Firebase.initializeApp(), dotenv load
├── firebase_options.dart    # Auto-generated by FlutterFire CLI (gitignored)
├── models/                  # Pure Dart data classes — fromJson/toJson only
│   ├── user_model.dart      # uid, displayName, bio, interests, avatarUrl, phone
│   ├── room_model.dart      # roomId, name, tags, creatorId, rules, memberCount
│   ├── message_model.dart   # msgId, senderId, text, timestamp, flagged
│   ├── review_model.dart    # reviewId, authorId, targetId, rating, body
│   └── report_model.dart    # reportId, reporterId, targetId, reason, timestamp
├── services/                # All external I/O — no BuildContext, no widgets
│   ├── auth_service.dart    # signIn, signUp, signOut, verifyPhone, authStateChanges
│   ├── firestore_service.dart  # CRUD + real-time stream wrappers
│   ├── storage_service.dart    # uploadAvatar, getDownloadUrl
│   ├── gemini_service.dart     # moderateMessage(text, roomRules), detectCommentSource(text)
│   └── report_service.dart     # submitReport, listReports
├── providers/               # ChangeNotifier classes consumed by UI via Provider
│   ├── auth_provider.dart   # Current user state; wraps AuthService
│   ├── room_provider.dart   # Room list, active room, member stream
│   └── profile_provider.dart   # Viewed profile, review list, aggregate rating
├── screens/                 # One subfolder per feature area
│   ├── auth/                # login, register, phone_verify
│   ├── home/                # BottomNavigationBar scaffold
│   ├── rooms/               # room_list, room_detail, create_room
│   ├── chat/                # chat_screen — StreamBuilder on messages; calls GeminiService pre-send
│   ├── profile/             # profile, edit_profile, reviews
│   └── report/              # report_screen
├── widgets/                 # Reusable components
│   ├── chat_bubble.dart     # Sent vs. received; flagged-message banner
│   ├── room_card.dart       # Tag chips + member count
│   ├── review_card.dart     # Stars + body + edit/delete menu
│   ├── star_rating_bar.dart # Interactive 1–5 star input
│   ├── interest_chip.dart   # Selectable tag chip
│   ├── avatar_picker.dart   # Image pick → StorageService upload
│   └── report_button.dart   # Wired to ReportScreen
├── router/
│   └── app_router.dart      # Named routes, auth guard, deep-link support
└── utils/
    ├── validators.dart      # Email, password, OTP validation
    └── constants.dart       # Firestore collection names, Gemini model name, defaults
```

## Key design rules

- **Services are widget-free.** No `BuildContext`, no Flutter widgets in `services/`. They return plain Dart types or streams.
- **Providers are the only bridge** between services and the UI. Screens call provider methods, never services directly.
- **Models are plain Dart.** No Flutter imports in `models/`. Only `fromJson`/`toJson` and simple getters.
- **Gemini is called pre-send.** `chat_screen.dart` calls `GeminiService.moderateMessage()` before writing to Firestore. Blocked messages never reach the database.
- **`lib/firebase_options.dart` is gitignored.** Regenerate it with `flutterfire configure` after cloning.
- **`.env` is a Flutter asset** (declared in `pubspec.yaml`). Load it at app start in `main.dart` before Firebase init.
