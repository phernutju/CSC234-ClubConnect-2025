# ClubConnect
A real-time, drop-in community app where users host and join hangout chat rooms based on shared interests — with AI-powered content moderation, a reputation-driven review system, and SMS-verified accounts.

Built for CSC231 (Agile Software Engineering) and CSC234 (Mobile Application Development) at School of Information Technology, KMUTT.

# Feature Core
| Feature | Description |
|---|---|
| **Community Rooms** | Create, browse, and join chat rooms filtered by interest tag |
| **Real-time Chat** | Live message stream via Firestore listeners; auto-scroll to latest message |
| **Authentication** | Email/password registration and login via Firebase Auth |
| **Phone Verification** | SMS OTP verification via Firebase Phone Auth to confirm account identity |
| **User Profile** | Edit avatar (Firebase Storage), username, bio, and personal interest tags |
| **Peer Review System** | Write, edit, and delete text reviews on any user's public profile |
| **Star Rating** | Rate other users (1–5 stars) after interaction; aggregate score shown on profile |
| **Home & Navigation** | Bottom navigation bar linking all core sections from a central home screen |
| **AI Content Moderation** | Each message is evaluated by Gemini against the room's own rules before it is displayed; inappropriate text, emoji abuse, and policy violations are flagged and blocked automatically |
| **Comment Source Detection** | Gemini analyses message content and context to identify which room or community a comment most likely originates from, surfacing cross-community behaviour |
| **Delete Comments** | Room creators and message authors can permanently delete any message |
| **Report System** | Users can report a message or profile; reports are stored in Firestore for admin review |

## Setup & Run

### 1. Clone the repository

```bash
git clone https://github.com/your-org/clubconnect.git
cd clubconnect
```

### 2. Install Flutter dependencies

```bash
flutter pub get
```

### 3. Run on Android
```bash
flutter run
```

### 4. Run on Web
```bash
flutter run -d chrome
```

### 5. Connect Firebase to the app

```bash
firebase login
flutterfire configure
```

Select your Firebase project and the platforms you need (Android, iOS). This command generates `lib/firebase_options.dart` — **do not commit this file** (already in `.gitignore`).

## Architecture

The project follows a layered architecture: **UI → Providers → Services → Firebase / Gemini**.

No screen imports Firebase or Gemini directly — all external calls go through the service layer.

```
lib/
├── main.dart                          # App entry point; Firebase.initializeApp(), dotenv load
├── firebase_options.dart              # Auto-generated by FlutterFire CLI (gitignored)
│
├── models/                            # Pure Dart data classes — fromJson / toJson only
│   ├── user_model.dart                # uid, displayName, bio, interests, avatarUrl, phone
│   ├── room_model.dart                # roomId, name, tags, creatorId, rules, memberCount
│   ├── message_model.dart             # msgId, senderId, text, timestamp, flagged
│   ├── review_model.dart              # reviewId, authorId, targetId, rating, body
│   └── report_model.dart              # reportId, reporterId, targetId, reason, timestamp
│
├── services/                          # All external I/O — no BuildContext, no widgets
│   ├── auth_service.dart              # signIn, signUp, signOut, verifyPhone, authStateChanges
│   ├── firestore_service.dart         # CRUD + real-time stream wrappers for every collection
│   ├── storage_service.dart           # uploadAvatar, getDownloadUrl
│   ├── gemini_service.dart            # moderateMessage(text, roomRules), detectCommentSource(text)
│   └── report_service.dart            # submitReport, (admin) listReports
│
├── providers/                         # ChangeNotifier classes; consumed by the UI via Provider
│   ├── auth_provider.dart             # Current user state; wraps AuthService
│   ├── room_provider.dart             # Room list, active room, member stream
│   └── profile_provider.dart          # Viewed profile, review list, aggregate rating
│
├── screens/                           # One subfolder per feature area
│   ├── auth/
│   │   ├── login_screen.dart
│   │   ├── register_screen.dart
│   │   └── phone_verify_screen.dart   # OTP input and resend logic
│   ├── home/
│   │   └── home_screen.dart           # BottomNavigationBar scaffold
│   ├── rooms/
│   │   ├── room_list_screen.dart      # Tag filter bar + room grid
│   │   ├── room_detail_screen.dart    # Room info, join/leave
│   │   └── create_room_screen.dart    # Name, description, tags, moderation rules form
│   ├── chat/
│   │   └── chat_screen.dart           # StreamBuilder on messages; calls GeminiService pre-send
│   ├── profile/
│   │   ├── profile_screen.dart        # Avatar, bio, star rating summary, review list
│   │   ├── edit_profile_screen.dart   # Avatar picker + form fields
│   │   └── reviews_screen.dart        # Full review list; create / edit / delete
│   └── report/
│       └── report_screen.dart         # Reason picker + optional detail text
│
├── widgets/                           # Reusable stateless / stateful components
│   ├── chat_bubble.dart               # Sent vs. received styling; flagged-message banner
│   ├── room_card.dart                 # Room tile with tag chips and member count
│   ├── review_card.dart               # Star display + review body + edit/delete menu
│   ├── star_rating_bar.dart           # Interactive 1–5 star input widget
│   ├── interest_chip.dart             # Selectable tag chip used in profile and room forms
│   ├── avatar_picker.dart             # Image pick → StorageService upload trigger
│   └── report_button.dart             # Contextual report icon wired to ReportScreen
│
├── router/
│   └── app_router.dart                # Named routes, auth guard, deep-link support
│
└── utils/
    ├── validators.dart                # Email format, password strength, OTP length checks
    └── constants.dart                 # Firestore collection names, Gemini model name, defaults
```
